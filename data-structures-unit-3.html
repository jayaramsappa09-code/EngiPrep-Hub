01

Last-In-First-Out (LIFO) Stack Operations

A Stack is a linear structure conforming to the Last In First Out (LIFO) operational principle. Elements are pushed onto and popped from the absolute top boundary exclusively.

  • Push: Adds an element onto the top.
  • Pop: Removes and returns the element at the top.
  • Peek/Top: Audits the value of the top element without altering structure states.
02

Array vs List Stack Implementations

Stacks can be implemented dynamically or sequentially with different boundaries:

Arrays Implementation

Uses a fixed size sequence array and an integer tracking index (top). Quick constant access, but risks throwing an Overflow error when memory boundaries are exceeded.

Linked List Implementation

Pushes and pops elements from the head boundary of a Singly Linked List, where the head represents the active top pointer. Completely avoids overflow issues because it allocates node memory dynamically.

03

Essential Overflow Bounds

Overflow Conditions

Triggered in array-backed stacks when pushing onto a completely saturated index: top == MAX - 1.

Underflow Conditions

Triggered during pop requests on an empty stack structure: top == -1.

04

Core Array-Backed Stack Push/Pop Code

Write simple array-backed stack structures containing boundaries checks in standard C:

#define MAX 100
int stack[MAX];
int top = -1;

// Push operation
void push(int val) {
    if (top == MAX - 1) {
        printf("Overflow Error\n");
        return;
    }
    stack[++top] = val; // Increment top index FIRST, then assign value
}

// Pop operation
int pop() {
    if (top == -1) {
        printf("Underflow Error\n");
        return -1;
    }
    return stack[top--]; // Return top value FIRST, then decrement index
}
05

High-Yield Stack Cheat Sheet

Quick operational cost breakdown for Stack data types:

Push & Pop operational cost

Time complexity: $\mathcal{O}(1)$

We push or pop elements from the top index exclusively; no shifting is required.

Element Access / Search Cost

Time complexity: $\mathcal{O}(n)$

To search inside a stack, we must systematically pop and examine every element.

JNTUK Solved Evaluation paper

Evaluate Postfix Equation "2 3 1 * + 9 -" (5 Marks JNTUK Board)

To evaluate a postfix string, push values onto the stack. When an operator is read, pop the top two values, apply the operator, and push the result back:

1. Data 2: Push(2) -> Stack: [2]

2. Data 3: Push(3) -> Stack: [2, 3]

3. Data 1: Push(1) -> Stack: [2, 3, 1]

4. Operator '*': Pop(1), Pop(3). 3 * 1 = 3. Push(3) -> Stack: [2, 3]

5. Operator '+': Pop(3), Pop(2). 2 + 3 = 5. Push(5) -> Stack: [5]

6. Data 9: Push(9) -> Stack: [5, 9]

7. Operator '-': Pop(9), Pop(5). 5 - 9 = -4. Push(-4) -> Final Result: -4

LIFO TRAY STACK MNEMONIC
