Supabase Setup Guide

1. Create a Supabase Project

Go to supabase.com, create a free project, and wait for it to provision.

2. Copy API Keys

In your Supabase Dashboard, go to Project Settings > API. Copy the following:

  • Project URL → Save as VITE_SUPABASE_URL
  • anon public (API Key) → Save as VITE_SUPABASE_ANON_KEY

3. Run Database Setup (SQL)

In Supabase, go to the SQL Editor, create a new query, and paste the contents of the database.sql file found in this project. Run it to create the necessary tables and triggers.

4. URL Configuration (Important)

In Supabase Authentication > URL Configuration, set the Site URL and Redirect URLs to your current app URL to ensure login and confirmation links work:

https://ais-dev-5t2imy5wglvbwi3ebzlli7-37198302098.asia-southeast1.run.app

5. Disable Email Confirmation (Optional)

If you want users to log in instantly without checking email: Go to Authentication > Providers > Email and toggle off "Confirm email". (Recommended for testing)

6. Enable Google Login (Optional)

To use Google Sign-In, go to Authentication > Providers in Supabase and enable Google. You will need to create a project in Google Cloud Console to get a Client ID and Secret.

Note for Email Signup:

By default, Supabase sends a confirmation email. You must click the link in your email to log in for the first time.
