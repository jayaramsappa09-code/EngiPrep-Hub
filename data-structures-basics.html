Introduction, Memory Coherence & Pointers

All program execution relies on memory organization. An Abstract Data Type (ADT) defines a mathematical model for a data type along with the operations allowed on it, entirely separating the logical definition from its physical memory implementation.

Segmented Memory Anatomy:

• The Stack Frame: Handles local dynamic parameters and automatic function activation blocks. Storage runs in strict, highly fast LIFO order managed automatically by the frame pointer.

• The Heap Space: Serves unstructured dynamic storage pools requested explicitly by the user (C: malloc, C++: new). Must be hand-freed to avoid fatal leaks.

To represent these dynamic heap links in native code, look at this structured layout formula:

Key Core Principles & Complexity

Structural Sorting & Bounds

Syllabus Classification Trees

1. Linear Structures

Elements are arranged in a sequential, adjacent format where each cell has a distinct unique predecessor and successor (Arrays, Lists, Stack, Queue).

2. Non-Linear Structures

Hierarchical or mesh arrangements that form complex multidirectional relationships rather than simple single chains (Trees, Graphs).

Asymptotic Complexity Metrics:

Determined as input sizes approach infinity ($n \to \infty$).

  • Big-O notation ($O$): Formally $f(n) = O(g(n))$ if there exist positive constants $c, n_0$ such that $0 \le f(n) \le c \cdot g(n)$ for all $n \ge n_0$. Describes the absolute worst-case time bounding limit.
  • Big-Omega ($\Omega$): Represents $f(n) = \Omega(g(n))$ where $f(n) \ge c \cdot g(n)$ for all $n \ge n_0$. Describes the lower bounds or best-case behavior.
  • Big-Theta ($\Theta$): Describes tight bounding, meaning $c_1 \cdot g(n) \le f(n) \le c_2 \cdot g(n)$ showing precise growth traits.

Essential Definitions

C STRUCT TEMPLATE

Self-Referential Structure

A struct in C or C++ that contains a pointer member directing to a structure variable of the same type. Absolutely essential for forming dynamic node-link structures. 

struct Node {
    int data;
    struct Node* next; // Self-pointer
};
TOPPER DICTIONARY

Syllabus Core Definitions:

  • ⚡ Array ADT: Contiguous, sequential memory slots that host uniform data arrays indexed with Instant Access $O(1)$.
  • ⚡ Singly Linked List: Series of nodes linked sequentially via single forward direction pointers.
  • ⚡ Sizeof pointer: Standard pointer addresses require exactly 4 bytes in 32-bit compilation, and 8 bytes in 64-bit platforms.

Core Board Derivations (JNTUK Heavy Weight)

Derivation A

Average Case Time Complexity & Recurrence Case of Quick Sort

Quick Sort splits around a chosen Pivot element. Assuming equal probability ($1/n$) of the pivot landing on any relative position index, we represent the recurrence relation as:

$$T(n) = \Theta(n) + \frac{2}{n} \sum_{i=1}^{n-1} T(i)$$ Multiply by $n$ to clear the denominator: $$n T(n) = \Theta(n^2) + 2 \sum_{i=1}^{n-1} T(i)$$ Subtracting the term for $(n-1)$ to perform mathematical reduction: $$n T(n) - (n-1)T(n-1) \approx 2 T(n-1) + 2n$$ Dividing by $n(n+1)$ and applying standard summation yield the result: $$T(n) = 2(n+1) \ln(n) \implies \Theta(n \log n)$$

⚠️ Exam Alert: Shows why unbalanced $k=0$ bad splits yield $O(n^2)$, whereas uniform splits yield $O(n \log n)$.

Derivation B

Circular Queue Boundary Pointer modulo Equations

In a linear queue, when elements are dequeued, the vacated indices at the front cannot be reused. This produces a "False Overflow" error despite empty capacity. Modulo arithmetic wraps indexes around boundaries.

Rear index increment: rear = (rear + 1) % MAX
Front index increment: front = (front + 1) % MAX
Circular Queue FULL condition: (rear + 1) % MAX == front
Derivation C

Minimum and Maximum Height Limit of Binary Search Tree ($N$ Nodes)

Let $h$ denote height, and $N$ be total nodes:

• Skewed Tree (Max Height): Each non-leaf node has exactly one child. $$h_{max} = N - 1 \implies O(N) \text{ search latency.}$$ • Perfect Balanced Tree (Min Height): All levels completely filled. $$N = 2^{h+1} - 1 \implies h_{min} = \lfloor \log_2(N+1) \rfloor - 1 \implies O(\log N) \text{ search.}$$

High-Yield Formula Sheet

JNTUK R23 semester exams frequently feature numerical questions on mapping multidimensional arrays to physical memory addresses. Use these exact formulas.

Row-Major Order Mapping (RMO)

2D Array Address (Row Major)

Address of $A[i][j]$: $$addr = Base + [i \times C + j] \times Size$$ Where $C$ is the total Column Count, and $Size$ is element size in bytes.
Column-Major Order Mapping (CMO)

2D Array Address (Column Major)

Address of $A[i][j]$: $$addr = Base + [j \times R + i] \times Size$$ Where $R$ is total Row Count of the matrix interface structure.

Memory Aids & Mnemonics

VERBAL SYMBOLS MATRIX

1. The STACK Mnemonic: "LIFO Card Trays"
Imagine a dinner tray stack. You place ("Push") onto the top, and remove ("Pop") only from the top. The first tray placed rests at the absolute bottom, yielding Last In First Out memory behavior.

2. The QUEUE Mnemonic: "FIFO Checkout Gates"
Imagine a grocery store checkout lane. Customers join at the rear ("Enqueue") and exit after paying at the front ("Dequeue"). Strict First In First Out behavior.

3. Singly traversal safety rule: "No Null Pointer Jumps"
Never write temp = temp->next inside loops without checking temp != NULL first. Otherwise, your program will crash instantly from a Segmentation Fault!
