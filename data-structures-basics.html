Core Concepts Library

Concept 1

Abstract Data Type (ADT) Parameters

An ADT defines a user-facing data representation and the set of operations permitted on it, completed without exposing the underlying implementation. It serves as an architectural design blueprint (independent of any programming language).

Concept 2

Time & Space Complexity Metrics

A standardized mathematical framework to analyze algorithmic efficiency. Time Complexity calculates the instruction counts as input size grows toward infinity, while Space Complexity quantifies auxiliary RAM overhead.

Program Synthesis: Linear & Binary Search

Binary Search C/C++ Implementation (Standard Iterative)

Time: O(log n) • Space: O(1)

Real-world Analogy: Locating a specific contact inside a physical dictionary by jumping immediately to the middle, checking ordering, and halving the target search area. Requires pre-sorted arrays. 

int binarySearch(int arr[], int size, int target) {
    int low = 0, high = size - 1;
    while (low <= high) {
        int mid = low + (high - low) / 2;
        if (arr[mid] == target) 
            return mid; // Target index located!
        if (arr[mid] < target) 
            low = mid + 1; // Discard left half
        else 
            high = mid - 1; // Discard right half
    }
    return -1; // Target is absent
}
Step Low Index High Index Calculated Mid (Val) Decision Outcome
1 0 (Val: 12) 8 (Val: 90) 4 (Val: 56) Target == 56. Located at index 4!

Sorting Architectures Deep-Dive

Bubble Sort Strategy

Repeatedly compares adjacent cells and swaps them if disordered. Sinks largest elements to the end of array first.

Worst/Avg: O(n²) • Best: O(n) (sorted)
Stability: Stable • In-Place: Yes
for (i = 0; i < n-1; i++)
  for (j = 0; j < n-i-1; j++)
    if (a[j] > a[j+1]) swap();

Selection Sort Strategy

Divides array into sorted and unsorted zones. Iteratively selects the absolute minimum from unsorted and places it at the start.

Worst/Avg/Best: O(n²) complexity
Stability: Unstable • In-Place: Yes
for (i = 0; i < n-1; i++) {
  min = i;
  /* Locate unsorted min & swap */
}

Insertion Sort Strategy

Loops through from index 1. Inserts the active target card into its precisely sorted correct slot in the left sorted slice.

Worst/Avg: O(n²) • Best: O(n) (sorted)
Stability: Stable • In-Place: Yes
for (i = 1; i < n; i++) {
  key = a[i]; j = i-1;
  while (j >= 0 && a[j] > key)...
}

Metric Parameter Static Array Representation Dynamic Linked List

Singly Linked List Nodes creation & Position Insertion Complexities: Insert Head O(1) • Insert End O(n) 

#include <stdio.h>
#include <stdlib.h>

struct Node {
    int data;
    struct Node* next;
};

// Insert at the beginning (Head Insertion)
struct Node* insertAtHead(struct Node* head, int data) {
    struct Node* newNode = (struct Node*)malloc(sizeof(struct Node));
    newNode->data = data;
    newNode->next = head; // Points to current head
    return newNode;       // New node becomes the new head
}

// Pointer structure flow:
// 1. Create newNode: [data|next]
// 2. Map newNode->next to head: [data|next] ---> [HeadNode|...]
// 3. Update Head reference to active newNode.

Common Student Trap: Forgetting to allocate heap memory using (struct Node*)malloc(...) in C. If you merely declare the reference pointer, writing to it yields an immediate Segmentation Fault!

Unit III Syllabus Focus

Stacks Introduction, Array/List implementation, & Expression evaluation

Understand the properties of LIFO. Build Stack arrays and linked implementations. Discover Expression evaluations (Infix, Postfix, Prefix) and Backtracking traversal paths.

Stack Operations Roadmap

  • 1. Push Functionality: Add data to top. Verify top == MAX - 1 overflow condition for solid arrays.
  • 2. Pop Functionality: Retrieve and remove. Double check top == -1 underflow blockages.
  • 3. ADT Infix Conversions: Maintain compiler tracking array grids using standard precedence weights.

Checklist

Method A: Static Stack Array Structure

Simple index array operations. The integer variable top points to the active element. Inserts require incrementing top; removals require decrementing top.

top++ --> pushes val to stack top-- --> pops val off stack Overflow condition: top == Limit

Method B: Linked List Stack (Dynamic Stack)

No size limits! Memory is dynamic. The active top pointer points to the head node of the list. Pushing is equivalent to adding a node at the head, and popping is removing the head node.

push: temp->next = top; top = temp; pop: temp = top; top = top->next; free(temp);
Unit IV Syllabus Focus

Queues & Deques Introduction, Circular Pointers and operations

Deep dive into FIFO (First In First Out) concepts. Construct Linear Queues, Circular indexes, Double-Ended Queues (Deques), and trace queue applications such as Breadth-First-Search traversals.

Unit IV Learning Roadmap

  • 1. Front/Rear Movement: Rear adds items (Enqueue); Front retrieves them (Dequeue).
  • 2. Circular Index wrap-around: (rear + 1) % MAX. Solves unused slots error in linear array indices!
  • 3. Double Ended Queue: Permitting operations at both ends.

Syllabus Check

Circular Queue Index Optimization Formulas

In a standard linear queue, once elements are dequeued, the starting cells remain completely empty and cannot be reused, triggering a premature "Overflow" error. Circular Queues solve this using modulo index math:

Check Empty
front == -1
Enqueue rear index increment math
rear = (rear + 1) % MAX
Check Full Queue condition
(rear + 1) % MAX == front
Unit V Syllabus Focus

Trees, Hashing algorithms, & Graph representations

Master Binary Search Trees (BST traversal algorithms: pre-order, post-order, in-order). Analyze Hashing, Collisions (Chaining & Open Addressing), Graphs, and DFS/BFS traversals.

Unit V Study Sequence

  • 1. BST Insertions: Traverses left if smaller; traverses right if larger. Runs in O(log n) average.
  • 2. Hash Functions: Module arrays mapped using key mod array_size tables.
  • 3. Traversal systems: Depth First Search (DFS uses Stack) and Breadth First Search (BFS uses Queue).

Checklist

Binary Tree Traversals Paths Summary

L = Left, R = Right, N = Node Node
1. Inorder (L-N-R) traversal

Visits left child first, then active root parent node, then right child. Yields ascending sorted keys in BSTs!

2. Preorder (N-L-R) traversal

Visits active parent node immediately, then left, then right. Great for cloning nested tree setups.

3. Postorder (L-R-N) traversal

Visits left child first, then right child, then climbs to the root node. Used for file deletion tree cleans.

Resolution A

Chaining Method (Open Hashing)

Each index cell of the hash table points directly to an external Singly Linked List of collided elements. If collisions strike an identical cell, we simply append the node to the linked list.

No table size limit overflow! Worst case search complexity degrades to O(n) if all elements cluster at a single index cell.

Resolution B

Open Addressing (Closed Hashing)

All keys are stored squarely within the table itself. If a collision arises, we probe adjacent cells sequentially based on a formula:

  • Linear Probing: Probe is $h(x, i) = (h(x) + i) \pmod m$.
  • Quadratic Probing: Probing increments quadratically: $i^2$.
  • Double Hashing: Uses a secondary hash function for probe dynamic intervals.

Why is Binary Search faster than Linear Search logic?

Category: Search Efficiency
Topper Logic Verified

Linear search checks iteratively sequentially, $O(n)$. Binary Search halves the input size in every step, reducing operations logarithmic scale $O(\log n)$. For a 1 million elements list, Linear checks 1 million times, Binary only 20!

Required condition: Input must be pre-sorted.

What represents the main error in a standard linear Queue array?

Category: Data Structure Traps
Topper Logic Verified

In standard linear queues, empty spaces left behind by deleted values cannot be reused as rear reaches MAX bounds blindly, triggering a fake Queue Overflow error. Circular Queue indices resolve this.

Fixed with: rear = (rear+1)%MAX index loops.
