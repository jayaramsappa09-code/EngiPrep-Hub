01

Singly Linked List Representation

A Singly Linked List (SLL) is a collection of nodes allocated dynamically in non-contiguous location buffers. Each node has two components:

  • Data Field: Holds the actual payload information (integer, character, object).
  • Next Pointer: Contains the reference/address link to the subsequent node in sequence. The final node next parameter is NULL.
02

DLL & Circular List logic

We address traverse limits by introducing bidirectional linkages and loops:

Doubly Linked Lists (DLL)

Each node holds two pointers: prev and next. This allows full bidirectional traversal, although it increases memory consumption slightly.

Circular Linked Lists (CLL)

The final node points directly back to the head node instead of NULL, forming a circular loop useful for scheduling patterns.

03

Essential Tradeoffs List

Array Properties

Contiguous memory. Constant time random access $\mathcal{O}(1)$. Shifting cost during insertion is $\mathcal{O}(n)$.

Linked List Properties

Dynamic non-contiguous nodes. Safe pointers redirection. Insertion at head is constant $\mathcal{O}(1)$. Access time is progressive $\mathcal{O}(n)$.

Core SLL insertions code

Write clean C-styled pointers redirection templates clearly showing node creation:

// SLL Node structure
struct Node {
    int data;
    struct Node* next;
};

// Insert at Head operation
struct Node* insertAtHead(struct Node* head, int value) {
    struct Node* newNode = (struct Node*)malloc(sizeof(struct Node));
    newNode->data = value;
    newNode->next = head; // Point new node next link to old head
    return newNode;       // Return new node as updated head
}

High-Yield SLL Formula sheet

Operational complexity summaries for various parts of a Linked List:

Insertion/Deletion at Head

Time complexity: $\mathcal{O}(1)$

SLL, DLL, and CLL require only dynamic pointer flips to update the head node.

Pointer Search Traverse

Time complexity: $\mathcal{O}(n)$

To search or access an element, we must traverse the list sequentially from the head.

Dynamic Lists Solved Examples

BOARD SOLVED TEST: REDIRECT FLIPS FOR SLL REVERSAL

Trace pointer redirections step-by-step to reverse a list: Head -> [10] -> [20] -> [30] -> NULL.

1. Set three pointers: prev = NULL, current = head, next = NULL.
2. In loop, save next reference: next = current->next.
3. Flip pointers linkage: current->next = prev.
4. Progress trackers: prev = current, current = next.
5. When current equals NULL, make head = prev. Result linked: Head -> [30] -> [20] -> [10] -> NULL!

JNTUK Solved Papers DLL

Doubly LinkedList Insertion at Head in C (10 Marks JNTUK Exam)

To safely insert a new node at the head in DLL, update both the forward and backward pointers:

struct Node* insertHeadDLL(struct Node* head, int val) {
    struct Node* temp = (struct Node*)malloc(sizeof(struct Node));
    temp->data = val;
    temp->prev = NULL;
    temp->next = head;

    if (head != NULL) {
        head->prev = temp; // Link backward pointer of old head to temp
    }
    return temp; // Updated Head node returning
}

11

SLL vs Array cheats

Evaluate these comparison vectors:

  • Access Speed: Arrays: constant $\mathcal{O}(1)$ via index calculations. Lists: sequential traverse $\mathcal{O}(n)$.
  • Dynamic Allocations: Arrays can cause buffer overflow or waste unused pre-allocated indices. Lists allocate nodes strictly when needed.

Memory Aids & Mnemonics

SLL LINK MNEMONIC

"Flipping head requires connecting references before breaking links!"
In SLL insert actions, always assign newNode->next = head FIRST, then assign head = newNode. Reversing this order breaks your references chain, leaving the existing nodes orphaned in memory!
