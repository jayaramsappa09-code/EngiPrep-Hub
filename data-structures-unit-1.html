Linear Structures Introduction

A Linear Data Structure organizes items sequentially, forming an uninterrupted sequence. Each item (except the boundary first and last elements) has an exclusive predecessor and successor.

  • Continuous Sequence: Arrays represent linear configurations in physical contiguous memory.
  • Logical Sequence: Linked Lists represent linear setups connected dynamically via pointer references in non-contiguous paths.
  • Restricted Operations: Stacks (LIFO) and Queues (FIFO) apply strict rules on traversal and modification indexes.
02

Abstract Data Types (ADTs)

An Abstract Data Type (ADT) specifies the mathematical model and operations of a structure. It defines what operations are available, completely independent of how they are implemented.

The User Contract

Specifies interfaces (e.g. insert(), delete()) and expected behavior parameters. The underlying algorithms remain black-box details to client services.

Direct Standard Implementations

List ADTs can be backed dynamically by physical arrays, singly links, or double links, preserving API contracts but varying in execution cost metrics.

03

Asymptotic Complexity Metrics

Big-Oh ($\mathcal{O}$) Complexity

Represents the asymptotic upper bound (worst-case execution path). Formally: $f(n) \le c \cdot g(n)$ for all $n \ge n_0$.

Space Complexity

The total physical memory footprint required relative to input bounds. Focuses extensively on auxiliary recursion stack requirements.

Core Board Derivations

DERIVATION 1: BINARY SEARCH RECURRENCE RELATION

At each step, Binary Search limits the remaining workspace area by exactly half. The resulting recurrence representation for worst-case comparisons is: $$T(n) = T\left(\frac{n}{2}\right) + c \quad (T(1) = 1)$$

Applying the substitution method progressively: $$T(n) = T\left(\frac{n}{4}\right) + c + c = T\left(\frac{n}{4}\right) + 2c$$ $$T(n) = T\left(\frac{n}{2^k}\right) + k \cdot c$$

Unrolling until boundary terminates when $\frac{n}{2^k} = 1 \implies k = \log_2 n$: $$T(n) = T(1) + \log_2 n \cdot c \implies T(n) = \mathcal{O}(\log_2 n)$$

DERIVATION 2: MERGE SORT RECURSIVE COMPLEXITY

Split the array into two halves, sort recursively, and merge them back linearly: $$T(n) = 2T\left(\frac{n}{2}\right) + \mathcal{O}(n)$$

Applying Master Theorem Case 2 where $a = 2$, $b = 2$, and $f(n) = \mathcal{O}(n)$: $$\log_b a = \log_2 2 = 1 \implies f(n) = \Theta(n^{\log_b a}) \implies T(n) = \mathcal{O}(n \log n)$$

Recursive Binary Search C-Function Implementation (10 Marks JNTUK Board)

Ensure you write simple parameters and properly define the termination condition:

int recursiveBinarySearch(int arr[], int low, int high, int x) {
    if (low > high) return -1; // Boundary Terminated
    int mid = low + (high - low) / 2;
    if (arr[mid] == x) return mid;
    if (arr[mid] > x) return recursiveBinarySearch(arr, low, mid - 1, x);
    return recursiveBinarySearch(arr, mid + 1, high, x);
}

Quick Revision Sheets

Evaluate these three properties during revisions:

  • Bubble Sort: Stable sort, $\mathcal{O}(n)$ best-case with early termination flags.
  • Insertion Sort: Best fit for nearly-sorted files, online-capable sorting properties.
  • Merge Sort: Stable, highly predictable $\mathcal{O}(n \log n)$ time, but requires auxiliary workspace $\mathcal{O}(n)$.

Memory Aids & Mnemonics

MNEMONIC: BINARY SPLITTING

"Divide, Divide, Decimate Bounds!"
In Binary search, never search linear zones. Divide left, divide right, and locate indices bounds. Keep low = mid + 1 or high = mid - 1 to maintain log-time execution paths!
