01

Trees & Traversals

A Tree is a non-linear, hierarchical structure representing nodes linked by edges. A Binary Tree constrains every node to hold at most two children: Left Child and Right Child.

  • Preorder (DLR): Visit Root node first, then traverse left tree, then traverse right tree recursively.
  • Inorder (LDR): Traverse left tree, visit root node, then traverse right tree.
  • Postorder (LRD): Traverse left tree first, then right tree, and visit root node last.
02

BST and Hashing Logic

Non-linear organization formats data logically for search operations:

Binary Search Trees (BST)

Enforces a strict relational constraint: for any node, $ \text{Left\_Child} < \text{Root} < \text{Right\_Child} $. Inorder traversal of a BST always yields sorted data.

Hashing Maps Representation

Uses a hash function, e.g., $ \text{index} = \text{key} \% \text{SIZE} $, to map key values directly to table addresses in average constant time $\mathcal{O}(1)$.

03

Key Terminologies

Graph Representations

Adjacency Matrix: A 2D array representation. Time complexity: constant check $\mathcal{O}(1)$, Space: quadratic $\mathcal{O}(v^2)$. Adjacency List: An array of linked lists, optimal for sparse graphs.

Graph Traversals (BFS & DFS)

Breadth First Search (BFS) uses a Queue to traverse layer-by-layer. Depth First Search (DFS) uses a Stack or recursion to traverse deep along a path first.

Core Tree Traversals Codes

Write simple recursive traversals algorithms inside standard C nodes setup:

struct Node {
    int data;
    struct Node* left;
    struct Node* right;
};

// Inorder LDR Traversal representation
void inorderTraversal(struct Node* root) {
    if (root == NULL) return;
    inorderTraversal(root->left);  // Left child recurse FIRST
    printf("%d ", root->data);     // Visit Root data value
    inorderTraversal(root->right); // Right child recurse LAST
}
05

High-Yield Formula Sheet

Quick operational cost breakdown for non-linear structures:

BST Insertion & Search

Average: $\mathcal{O}(\log n)$ • Worst: $\mathcal{O}(n)$

Worst case occurs if elements are inserted in sorted order, creating a skewed tree.

Graph BFS and DFS Traversals

Time: $\mathcal{O}(V + E)$ • Space: $\mathcal{O}(V)$

Traverses every vertex ($V$) and edge ($E$) systematically in linear time.

11

Collision Resolution Sheets

Evaluate these collision resolution techniques:

  • Open Addressing: Includes Linear Probing ($i + 1$) and Quadratic Probing ($i + i^2$). It probes the table for the next empty slot directly.
  • Separate Chaining: Each slot points to a linked list of collided keys, avoiding primary clustering.

Memory Aids & Mnemonics

TREE TRAVERSAL DLR MNEMONIC

"DLR - Do Leave Right now!"
In Preorder traversal: D comes first -> visit root Data first. Then visit Left subtree. Then visit Right subtree last. Remember this pattern for the exam!
