01

First-In-First-Out (FIFO) Queue Mechanics

A Queue is a linear structure based on the First In First Out (FIFO) principle. Elements are inserted at the back end (rear) and removed from the front boundary (front).

  • Enqueue: Appends a data value at the rear of the queue.
  • Dequeue: Pops and yields the item located at the front index.
  • Properties: Requires dual-pointers track management: front and rear.
02

Array vs List Queue Representations

Queues can be represented sequentially using arrays or dynamically using nodes:

Sequential Array Storage

Maintains dual indexes: front and rear. Access operations are constant-time $\mathcal{O}(1)$, but the fixed array size limits the queue's capacity.

Linked List Nodes Tracking

References both head (front) and tail (rear) nodes. Dynamically allocates nodes to prevent overflow bugs, ensuring clean insertions.

03

The False Overflow Problem

In standard linear queues, dequeuing elements shifts the front index forward, leaving the empty slots before it unusable. When rear == MAX - 1, the queue triggers an Overflow error on enqueues, even if the slots before the front index are free. This block of unused memory is known as a False Overflow.

Circular Array Solution

Circular queues use modular arithmetic to wrap pointers back to index 0: rear = (rear + 1) % MAX. This recycled indexing model completely prevents false overflows.

Circular Empty/Full Check Formulas

Empty: front == -1. Full: (rear + 1) % MAX == front.

04

Core Circular Queue Enqueue/Dequeue Code

Write robust C-styled implementations showing modular arithmetic index calculations precisely:

#define MAX 5
int queue[MAX];
int front = -1, rear = -1;

// Enqueue in Circular Queue
void enqueue(int val) {
    if ((rear + 1) % MAX == front) {
        printf("Queue Overflow Error\n");
        return;
    }
    if (front == -1) front = 0; // First element boundary init
    rear = (rear + 1) % MAX;    // Wrap index circularly
    queue[rear] = val;
}

// Dequeue in Circular Queue
int dequeue() {
    if (front == -1) {
        printf("Queue Underflow Error\n");
        return -1;
    }
    int data = queue[front];
    if (front == rear) { // Solo element popped: Reset queue state
        front = -1; rear = -1;
    } else {
        front = (front + 1) % MAX; // Progress circularly
    }
    return data;
}

High-Yield Queue Time metrics

Quick operational cost reference chart for Circular and Linear Queue structures:

Enqueue & Dequeue Operations

Time complexity: $\mathcal{O}(1)$

Pointers (front/rear) are updated directly using constant-time calculations.

Pointer Search Traverse

Time complexity: $\mathcal{O}(n)$

To search inside a queue, we must systematically scan from the front index to the rear index.

BOARD TRACE TEST: CIRCULAR ARRAY WRAPAROUND

Trace a circular queue of size 5 with elements: [empty, empty, 30, 40, 50], where front = 2 and rear = 4.

1. Current State: front=2, rear=4, elements=3.
2. Action: Enqueue(60) requested.
3. Wrap pointer: rear = (rear + 1) % 5 = (4 + 1) % 5 = 0.

FIFO BUS LINE MNEMONIC

"Join at Rear, exit from Front!"
