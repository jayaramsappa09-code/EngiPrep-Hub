Unit 1

Arrays, Searching & Sorting

Core Axioms & Theories

Arrays provide contiguous memory allocation ensuring $O(1)$ time complexity for index-based access, but insertion/deletion requires $O(n)$ shifting overhead.

Exam Callout: Time Complexity

Examiners frequently test Binary Search complexity derivation. Always mention that the search space halves each iteration causing $T(n) = T(n/2) + O(1)$, deriving to $O(\log n)$.

Mathematical Notation & Code Constraints

// Binary Search Iterative Template
int binarySearch(int arr[], int l, int r, int x) {
  while (l <= r) {
    int m = l + (r - l) / 2; // Prevents overflow
    if (arr[m] == x) return m;
    if (arr[m] < x) l = m + 1;
    else r = m - 1;
  }
  return -1;
}

Unit 3

Linked Lists (SLL, DLL, CLL)

Dynamic memory allocation eliminates resizing overhead by maintaining non-contiguous structural nodes coupled via pointers. Master the variations of Singly, Doubly, and Circular Linked Lists.

Singly Linked List

Contains a data field and a next pointer. Best for simple stacks/queues implementation where reverse traversal is unnecessary.

Doubly Linked List

Contains prev, data, and next pointers. Ideal for complex applications like browser history or caching mechanisms (LRU).

Node Insertion Logic (SLL)

// Inserting at the beginning of a Singly Linked List
void insertAtHead(Node** head_ref, int new_data) {
  Node* new_node = (Node*) malloc(sizeof(Node));
  new_node->data = new_data;
  new_node->next = (*head_ref);
  (*head_ref) = new_node;
}
Unit 4

Trees & Hashing

Binary Search Trees (BST) provide structural bounds guaranteeing $O(\log n)$ search space geometry under balanced configurations. Understand AVL rotations and Hashing collision resolution (Linear vs Quadratic Probing).

Crucial Exam Theory

Tree traversals (Inorder, Preorder, Postorder) are repeatedly asked for 5 marks. Remember: Inorder traversal of a BST always yields a sorted sequence.

Standard Inorder Traversal

// Left -> Root -> Right
void inorder(Node* root) {
  if (root != NULL) {
    inorder(root->left);
    printf("%d ", root->data);
    inorder(root->right);
  }
}
Unit 5

Graph Algorithms

Graphs map interconnected systems using vertices and edges. Focus entirely on Dijkstra's shortest path resolution, Minimum Spanning Tree (Prim's & Kruskal's), and BFS/DFS graph traversals.

Graph Representation Methods

  • Adjacency Matrix: A 2D array $V \times V$ where $A_{ij} = 1$ if edge exists. High space complexity $O(V^2)$, but $O(1)$ edge query.
  • Adjacency List: Array of linked lists storing neighbors. Lower space complexity $O(V + E)$, but slower $O(V)$ edge querying. Preferred for sparse networks.

Step 1 (2 Marks): Definition
A binary tree where left child nodes contain values smaller than the parent, and right child nodes contain values strictly greater.

Step 2 (4 Marks): Insertion Routine
Recurse until null. If smaller, go left. If larger, go right. Insert as leaf node.

Step 3 (4 Marks): Deletion Routine (3 Cases)
Case A: No children (Delete directly). Case B: One child (Bypass). Case C: Two children (Find Inorder Successor, swap context, delete leaf).

2 Marks Nov 2022

Differentiate between Linear Probing and Quadratic Probing in Hashing collision resolution.

Linear Probing: Looks for the next immediate empty slot sequentially $(h(k)+i) \bmod N$. Prone to primary clustering.
Quadratic Probing: Uses polynomial variance $(h(k)+i^2) \bmod N$ to space out probes and avoid primary clustering.

External Examiner High-Frequency Checklist

"Why do we use Circular Queues instead of normal Queues?"
Normal arrays suffer from spatial wastage when dequeuing elements (front pointer shifts and space behind it is permanently lost). Circular queues wrap the bounds using modulo offsets preventing memory exhaustion.
"What exactly is an Adjacency Matrix vs Adjacency List?"
Matrix is a 2D array representation providing $O(1)$ edge checking but costing $O(V^2)$ memory space (bad for sparse graphs). List is dynamic LinkedList pointers per vertex, saving massive memory boundaries for dense topologies.
